OneHealth is a unified, end-to-end encrypted healthcare management ecosystem designed to centralize medical records. It eliminates data silos between different healthcare providers by using a "National Cloud" sync architecture.

A centralized, role-based healthcare management platform connecting patients, doctors, and labs. Features end-to-end encrypted record sharing, AI-driven recovery tracking, and a "National Cloud" sync architecture for seamless medical data portability.


Architecture: Explain the Patient-Doctor-Lab triad.

Tech Stack: Node.js, MongoDB, Python (AI Analytics), and Vanilla JS/Tailwind for the frontend.

Security Features: Mention the Dynamic Access Code system and encrypted sessions.


Key Modules & Features:
Patient Portal: Features an "AI Recovery Progress" tracker, a secure Medical Repository, and a dynamic "Doctor Access Port" (6-digit OTP system) to grant temporary record access to physicians.

Doctor/Hospital Dashboards: Role-based access to view patient histories, clinical findings, and MRI reports with integrated AI Diagnostic Insights.

Diagnostic Hub (OneLab): A dedicated interface for labs to upload reports directly to a patient’s profile using a unique Health ID (UNHRS).

Security: Focused on end-to-end encryption and secure session management for HIPAA-compliant data handling.
